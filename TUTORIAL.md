# 🚀 Quick Setup Tutorial

This guide allows you to set up the **LLM DSL Information Extraction** system in a few minutes, from scratch.

## 📋 Overview

The setup process consists of **5 main steps**:

1. Create a Python virtual environment
2. Install project dependencies
3. Configure API keys (`.env`)
4. Build the Qdrant index
5. ✅ Setup complete! **You can start using the system**

**Estimated time:** 10-15 minutes (~5 min for index building)

---

## Step 1️⃣: Create a Virtual Environment

A virtual environment isolates project dependencies from your system Python.

**On Windows:**
```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

**On macOS / Linux:**
```bash
python3 -m venv .venv
source .venv/bin/activate
```

Once activated, you should see `(.venv)` at the beginning of your terminal line.

---

## Step 2️⃣: Install Dependencies

With the virtual environment activated, install all project dependencies:

```bash
pip install -r requirements.txt
```

**What gets installed:**
- **Embedders & vectorstore:** `fastembed`, `sentence-transformers`, `qdrant-client` (default), `faiss-cpu` (optional)
- **LLM APIs:** `mistralai`, `groq`, `anthropic`, `ollama` (for local models)
- **Utilities:** `python-dotenv`, `pyyaml`, `tiktoken`, `langgraph`
- **Benchmarking:** `scikit-learn`, `rich`, `typer`

> **Note:** If you encounter compilation issues, ensure you have Python 3.9+ installed.

---

## Step 3️⃣: Configure API Keys (.env)

### Copy Example File

```bash
# Windows
copy .env.example .env

# macOS / Linux
cp .env.example .env
```

### Fill in API Key

Open the created `.env` file and add your **DeepSeek** API key (the project's default model):

```yaml
# .env
DEEPSEEK_API_KEY=your_deepseek_api_key_here
```

**Where to get your DeepSeek key?**
1. Go to [https://platform.deepseek.com/](https://platform.deepseek.com/)
2. Create an account or log in
3. Go to "API Keys" → "Create New API Key"
4. Copy your key and paste it in `.env`

### (Optional) Other API Keys

If you want to test other LLM models, you can also add:

```yaml
MISTRAL_API_KEY=your_mistral_key_here
GROQ_API_KEY=your_groq_key_here
# etc.
```

But to get started, **only the DeepSeek key is required**.

---

## Step 4️⃣: (Optional) Configure config.yaml

The `config.yaml` file contains the system's default configuration. **The defaults work very well to get started**, but you can customize them if needed (LLM models, embedder type, vector database, etc.).

For a quick setup, you can **skip this step** and keep the default settings.

---

## Step 5️⃣: Build the Qdrant Index

The Qdrant index allows the system to quickly retrieve relevant information from DSL scripts.

```bash
python build_summary_index.py
```

**What happens:**
1. Parses `.nvn` files from the `env_scripts/` folder
2. Generates **summaries** (semantic summaries) for each code block
3. Embeds the summaries
4. Builds the Qdrant index in `data/qdrant/`

**Duration:** ~5 minutes (depends on number of scripts and connection speed)

> **Note:** If you want to use **FAISS** instead of Qdrant, configure `config.yaml` with `embedder.type: faiss` and run `python build_index.py` or `python build_summary_index.py` instead.

Once complete, you'll see a ✅ message confirming the index has been created.

---

## ✅ Setup Complete!

Congratulations 🎉! Your setup is now **complete and operational**.

You can start using the system with the main features (see next section).

---

## 🎯 Main Features

The system has **3 main usage modes**: Query, Benchmark, and Interactive.

### Query Mode: Query the System

Ask a single question and get an answer:

```bash
python main.py --query "Your question here"
```

**Examples:**
```bash
# Simple query
python main.py --query "How does the compilation process work?"

# With detailed output (trace reasoning step-by-step)
python main.py -v --query "What are the critical modules?"

# With minimal output (just final answer)
python main.py -q --query "List the global variables"
```

**Output:** The answer generated by the LLM, based on search in the index.

---

### Benchmark Mode: Evaluate Performance

Evaluate the system against a set of questions with reference answers:

```bash
python main.py --benchmarkpath test.json
```

**Variants:**
```bash
# Short benchmark file (recommended for testing)
python main.py --benchmarkpath test.json

# More complete file (warning: can take 10-30 min)
python main.py --benchmarkpath questions.json

# With detailed results
python main.py -v --benchmarkpath test.json

# Save results to JSON file
python main.py -sd --benchmarkpath test.json

# Resume from question N (useful if it crashes)
python main.py --benchmarkpath questions.json --benchmarkstart 5
```

**Output:** Performance score (e.g., cosine similarity, LLM judgment, hybrid score, etc.).

---

### Interactive Mode: Multi-Question Conversation

Launch an interactive session to ask multiple questions without restarting the script:

```bash
python main.py --interactive
```

**Example session:**
```
👤 Envision Copilot (Ctrl+C to exit)
User: What do the utility functions do?
Assistant: [answers your question]

User: Can you detail the data flow?
Assistant: [continues answering in same session]

User: quit
[End of session]
```

**Advantages:** Faster if you have multiple questions, index stays in memory.

## 🎛️ Options principales

Pour une liste **complète** de toutes les options, consulte l'aide intégrée :

```bash
python main.py --help
```

Ci-dessous, un tableau des options **les plus courantes** :

| Option | Court | Type | Description |
|--------|-------|------|-------------|
| `--interactive` | `-i` | flag | Lance le mode interactif (défaut) |
| `--query TEXT` | — | string | Lance une query unique puis quitte |
| `--verbose` | `-v` | flag | Affiche le flux de raisonnement détaillé |
| `--quiet` | `-q` | flag | Supprime les messages d'initialisation |
| `--agent NAME` | `-a` | choice | Change d'agent LLM : `mistral`, `groq`, `deepseek-v3`, `qwen`, `qwen-ssh` |
| `--indextype TYPE` | `-in` | choice | Type d'index : `full_chunk`, `summary` |
| `--querytransform MODE` | `-qt` | choice | Mode de transformation : `fusion`, `hyde`, `None` |
| `--benchmarkpath PATH` | `-bp` | path | Lance un benchmark avec les questions du fichier |
| `--benchmarktype TYPE` | `-bt` | choice | Type d'évaluation : `llm_as_a_judge`, `cosine_similarity`, `hybrid` |
| `--benchmarkagent AGENT` | `-ba` | choice | LLM agent for evaluation: `mistral`, `groq`, etc. |
| `--benchmarkstart N` | `-bs` | integer | Resume benchmark from question N |
| `--save_data` | `-sd` | flag | Save benchmark results as JSON |
| `--token_count` | `-tc` | flag | Show total tokens used |

**Combined examples:**
```bash
# Query with Mistral agent and fusion
python main.py --agent mistral --querytransform fusion --query "your query"

# Complete benchmark with detailed output
python main.py -v -bp questions.json -ba groq

# Interactive verbose with summary index instead of full_chunk
python main.py -i -v --indextype summary
```

## 🔧 Troubleshooting

### ❌ "ModuleNotFoundError" After Installation

**Solution:** Make sure the virtual environment is activated:
```bash
# Windows
.\venv\Scripts\Activate.ps1

# macOS / Linux
source venv/bin/activate
```

### ❌ "DEEPSEEK_API_KEY not found"

**Solution:** Verify that the `.env` file exists and contains your key:
```bash
# Check that file exists
cat .env  # macOS/Linux
type .env # Windows

# Check that key is present
grep DEEPSEEK_API_KEY .env
```

### ❌ "Index not found" When Launching

**Solution:** Build the index first:
```bash
python build_summary_index.py
```

### ❌ Index takes too long to build

**Solution:** This is normal for the first build (~5 min). For rebuilds, you can add `--rebuild` to force regeneration.

### ❌ Qdrant Connection Error

**Solution:** The default Qdrant configuration uses in-memory mode. If you're using an external Qdrant server, configure `config.yaml` with the correct coordinates.
